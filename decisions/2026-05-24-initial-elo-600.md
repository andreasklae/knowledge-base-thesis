# Initial agent ELO set to 600

## Context

The ELO system needs a starting value for the agent before any games are played. The original scaffold used 1200 — the standard USCF/Lichess default for "unknown skill." With the switch to Gemma 4 31B-it running via eX3 vLLM and no skill library yet, 1200 is too optimistic: prior testing with comparable models showed the agent losing its way down to the 700–800 range over the first 10–15 calibration games, wasting provisional K-factor games on a long downward drift.

## Decision

Set `INITIAL_ELO = 600` in `backend/app/elo.py`.

This is an educated guess based on observing gameplay during testing. The agent at this stage plays basic legal moves but makes obvious blunders — consistent with chess strength in the 500–700 range. Starting at 600 puts the prior closer to where the agent will actually land, reducing calibration waste without over-correcting below where it might realistically play.

This is documented as a guess, not a derived measurement. If Phase 1 calibration reveals the true level is above or below 600, the relevant comparison in the thesis is still the *relative* trajectory (Elo over games played), not the starting value.

## Consequences

- Fewer games are spent drifting downward during initial calibration.
- The initial ELO is lower than the conventional FIDE/USCF default; this must be disclosed in the methods section. The framing: "we set a model-informed prior rather than the conventional unrated default, to reduce calibration waste."
- If the agent's true level turns out to be above 600, it will climb from the start rather than fall — which is a better empirical story anyway.
- `agent_elo.json` was cleared when all prior game data was deleted (2026-05-24), so this takes effect from the first game of the new calibration run.
