# Initial agent ELO revised to 1200

## Context

Initial ELO was set to 600 (see superseded ADR) based on early gameplay
observations during testing. First calibration batch (vs chess.com 550)
produced games where the agent had clearly won positions (queen + king vs
lone king) but could not deliver checkmate within a reasonable number of
moves, resulting in move-cap draws. Calibration at 600 would require many
games of slow upward drift before reaching the agent's actual strength band.

## Decision

Reset `INITIAL_ELO` to 1200. This is the conventional default for an
unrated player and a more honest prior given that the agent — a 31B
instruction-tuned model with structured prompting — is likely to play at
a mid-amateur level rather than near-beginner. The 600 baseline produced
misleading calibration data and wasted provisional K-factor games.

`agent_elo.json` was wiped; all prior game data remains in
`games/experimental.csv` for forensic reference but does not affect ELO
going forward.

## Consequences

- Calibration starts closer to the agent's real strength band, so the
  provisional K=40 games deliver useful signal faster.
- The prior experimental.csv rows at ELO 600 are orphaned from the new
  baseline; they remain for audit purposes but are not cited in the thesis
  calibration results.
