# ADR: Agent-authored turn memory via --reasoning on make_move.py

**Date:** 2026-05-26  
**Status:** Accepted  
**Branch:** visualization-and-context-management

## Context

After implementing the HistoryProcessor-based context injection (see the per-turn-fresh-context ADR and the 2026-05-26-context-management diary entry), the turn memory itself was still generated by a throwaway LLM agent. That agent received only the committed UCI string and a list of rejected candidates — no board state, no position, no reasoning from the game agent's actual analysis.

The results were poor: hallucinated move intent, generic "Watch" text repeated across every turn, empty "Rejected" fields (the agent rarely called `imagine_move` before committing on retries). More critically, the summaries were often of the wrong move: the agent described one move in text on attempt 1, got a retry prompt with no prior context, committed a different move on attempt 2, and the throwaway summariser faithfully described the rushed second-attempt move.

## Decision

Remove the throwaway LLM summariser entirely. Add a required `--reasoning` parameter to `make_move.py`. The game agent writes its own note at the moment of commitment, in plain text after `--reasoning`, with full positional context available. The harness reads the reasoning from the success response JSON and stores it verbatim as the turn memory.

**`--reasoning` is required.** If omitted, `make_move.py` returns `ok=false` with an error and the move is not committed. The agent must retry the tool call with both arguments. This means:
- A move cannot be committed without a justification
- The justification is written when the agent has the most information (at commit time)
- The "described a move without using the tool" pattern is no longer possible — to produce a reasoning you must also produce a `--uci`

## Argument parsing

`argparse.REMAINDER` is used for `--reasoning` so the agent can write unquoted free text:

```
run_script("chess-player", "make_move.py", "--uci e2e4 --reasoning I played e4 to control center. Rejected d4 (slower). Watch: c5 push. Sequence: none.")
```

The SDK passes `args` through `shlex.split`; `REMAINDER` captures everything after `--reasoning` as a list and the harness joins it back to a string. No quoting needed.

## Format

No server-side format enforcement. The agent writes whatever is useful. Suggested content (from SKILL.md): what was played and why, what was rejected, one opponent threat to watch, any ongoing plan. Framed as a note to the agent's future self — it is the only context available on the next turn.

An LLM interpreting free text on the next turn is more robust than a rigid 4-line schema that the model may or may not comply with correctly.

## Consequences

**Removed:**
- `_compact_turn()` async function (LLM synthesis call)
- `_build_summariser()` (throwaway agent construction)
- `_SUMMARY_PROMPT_TEMPLATE` (4-line format template)
- The `re` dependency on `_compact_turn` (still needed for Gemma marker stripping)
- The async `await` at the commit point (synchronous now)

**Changed:**
- `_committed_move_from_result()` returns `(uci, reasoning)` tuple instead of `str`
- `make_move.py` success JSON adds `"reasoning"` field
- `make_move.py` exits nonzero with error JSON if `--reasoning` is absent/empty
- SKILL.md updated to document `--reasoning` and frame it as turn memory

**Unchanged:**
- `_pending_summary` mutable container
- `_make_turn_memory_processor` HistoryProcessor factory
- Injection mechanism: `(original_prompt, reasoning)` tuple stored and injected before next turn's `run_stream_events`

## Alternatives considered

**Keep throwaway summariser but give it the FEN and prior reasoning.** Would have improved quality by providing positional context. Rejected: still an extra LLM call, still indirect, still requires the agent to have committed a well-analysed move in the first place. The fundamental problem — summaries of rushed retry moves — would persist.

**Use the agent's last thinking event as the turn memory.** The thinking text is already grounded in the position. Rejected: the agent sometimes emits no thinking event (empty thinking buffer), and thinking is not tied to the committed move specifically — it may reflect analysis that was discarded.

**Keep the 4-line format, enforce it server-side.** Rejected: format enforcement adds friction without adding value. The structure (last move, rejected, watch, sequence) is useful guidance but an LLM can extract equivalent information from unstructured prose. Enforcement would cause spurious `ok=false` retries when the model writes slightly different line formats.
