# The Four-Part Framework Is Committed at the Extended-Description Stage

## Context

A design-science thesis (see [`2026-05-13-methodology-design-science.md`](2026-05-13-methodology-design-science.md)) faces a recurring methodological pitfall: framework substance gets retrofitted to whatever the experiments produced. Whatever survives the cases gets called "the framework," and the thesis no longer tests anything — it just describes what happened with framework-flavoured vocabulary.

## Decision

The four-part decomposition — **workspace, tools, skills, data**, as four affordances rather than a strict partition — is **committed at the extended-description stage**, before any of the six experiments runs. See [`../wiki/concepts/framework-four-components.md`](../wiki/concepts/framework-four-components.md).

Phase 2 (Jan–April 2027) **refines articulation, not substance**:

- Examples and edge cases are added based on what the experiments showed.
- Components that the experiments revealed to be more or less important than initially weighted are re-emphasised accordingly in how the framework is presented.
- The substance — what the four components *are* — is not retrofitted to results.

## What this rules out

- "Adding a fifth component" because some experiment doesn't fit cleanly into the existing four.
- Renaming or repurposing a component so that an unexpected result "always was part of" the framework.
- Removing a component because an experiment did not produce a clean effect on it.

The framework is allowed to be **wrong**. If an experiment produces a result the framework does not predict, that is a finding about the framework — recorded and discussed — not an instruction to silently revise the framework so the result fits.

## Consequences

- The Framework chapter writes against this constraint: it is the canonical statement of the framework, prepared independently of the experimental write-ups.
- Phase 2 work is articulation, not foundational. The literature review can be expanded; the conceptual structure cannot.
- Any future change to the framework's substance requires a superseding decision record citing this one.
