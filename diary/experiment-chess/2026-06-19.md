# 2026-06-19 — Basic-mate technique, the blunder gate, and the technique-vs-sacrifice boundary

*Entry written by Claude (Opus 4.8) at Andreas's request, summarising the work
on the `mating-patterns-and-strategy` branch from 2026-06-13 through 2026-06-19.
It picks up where [[diary/experiment-chess/2026-06-10]] left off: that entry
diagnosed "reaches won positions, can't finish"; this stretch is the campaign to
fix the finishing, plus a blunder gate that grew out of the smoke games. All of
it ran as experimental / puzzle games with Elo frozen — the ranked record still
ends at game 053 (2026-05-27), Elo 793.6. Nothing here is a ranked datapoint
yet.*

## Where this picks up

PR #2 (`mating-patterns-and-strategy`) merged on 06-15 carrying the 06-10
work — Capablanca ingestion, the nine wiki pages, the mate/draw radar,
structured turn memory, puzzle mode. Work then continued *on the same branch*
after the merge (it is currently 22 commits ahead of origin with an uncommitted
working tree). Two threads ran in parallel: tightening the **single-major mate
technique** (K+R, K+Q) until it actually converts, and a **blunder gate** that
came out of the 06-13/15 smoke games.

## Thread A — the blunder gate (06-14/15)

The 06-15 smoke games surfaced the failure that isn't about mating at all:
the agent drops material in the middlegame. Game 058 (`9b0d7590`, 0-1 vs
chesscom-1000) lost via two *defended-square* bad trades (9.Nd2??, 16.Nxd6??) —
"defended" is not "safe". That motivated a static-exchange-evaluation (SEE)
layer:

- `imagine_move` now gives **value-based exchange warnings** (`f7c5b2e`): it
  runs SEE so "the square is defended" is replaced by "you come out down
  material on this capture sequence."
- A **blunder gate** (`a8db9c0`, `8424f7f`, `f9b2375`) blocks single-square
  losing trades — minor-for-pawn drops, self-losing moves get a hard
  (unconfirmable) class — and warns on own-back-rank weaknesses. The ritual
  `use_skill` call was dropped.
- A regression suite was seeded against it: a Sonnet Ruy-Lopez `Nxe5` blunder
  (`f693e1a`) and two recorded gate-validation games vs Maia-1100 (`7c2356f`,
  `fda1496`).

Per the running memory note, SEE is deliberately left **ungated for the move
the agent ultimately plays** in some paths — the gate informs, it does not fully
play for the agent — to stay clean under the tool-fairness rulebook
([[2026-06-02-tool-fairness-rulebook]], [[tool-fairness]]).

## Thread B — making single-major mates actually convert (06-13 → 06-18)

This is the long grind. The 06-10 entry had K+Q-by-radar working but K+R and
the ladder still drawing. The arc:

- **Ladder root-cause (06-13/14).** The advisor had been *naming illegal or
  forbidden checks* (`03e867d`). Rewrote it to give recipe-level / principle
  advice with a board self-check rather than prescribed moves (`69ff304`,
  `40fdf67`), and added the finish-exception for the rook-beside-cornered-king
  trap (`328bfb9`). Fairness line held: the advisor states lines and squares,
  never the move.
- **Reduce to one rule (06-17/18).** After repeated failures (rook fleeing to
  the corner, king shuffling at mate-in-2), the whole single-major advisor was
  replaced with **Capablanca's one deterministic confinement rule** (`423d77e`),
  driven by a `confine_state` that ranks every legal move by a measure and
  reports whether the king or the major should move.
- **The metric had to go piece-specific** (`8bb4ac9` → `725f7b1` → `8dc0bef`).
  Pure box-greedy drew by repetition (it shrank the *measured* box while
  abandoning the cut that mattered). K+R settled on `(box area, then kings
  closer)` plus a tiebreak to **keep the rook far from the enemy king**
  (Capablanca again) — in a 40-seed sim that more than doubled the mate rate
  (8/40 → 19/40). K+Q is king-march-primary, so it uses `(kings closer, then
  enemy-king mobility)` instead; box-greedy stalled the queen and risked
  stalemate. A property test now *proves* following `confine_state` mates a
  running king in ≤40 plies with no repetition.

Honest limitation recorded in the commits: the K+R side-jump is **mitigated, not
eliminated** — when a perpendicular move uniquely minimises the box it is still
chosen. A complete "stay on the cutting line, advance one line at a time" fix has
resisted several attempts; per Andreas, left as a clean improvement rather than
over-engineered. The real agent already mates K+R in ~23 plies in some games;
this makes it more frequent. 312 tests pass.

Also: priority-0 interrupt when an enemy pawn is one push from promoting (it
would queen a new major and break the mating net — game `912d0f7f`), and
turn-prompt hygiene — the per-turn prompt now carries only bare facts (whose
move, FEN, legal moves, the agent's own plan), with *all* position analysis
moved into the tools (`chess__show_position`). The prompt no longer analyses the
position or hints moves.

## Thread C — real puzzle sourcing + the sweep (06-19)

To test mates honestly, two scripts were added (`50246d6`) that **do not invent
positions**: `fetch_mate_puzzle.py` pulls real Lichess puzzles by theme (mirrors
black-to-move ones so the white-only agent solves them) and verifies the mate in
python-chess; `gen_named_mate_puzzles.py` derives configuration mates
(arabian/anastasia/greco) from the Wikipedia archetype geometry and walks back
one move to a verified "white to move, mate in 1".

The mate-conversion sweep result (`2ef7479`) is the headline finding:

- **PASS — material + technique mates:** K+2R, K+R, K+Q, back-rank, arabian,
  anastasia, greco, hook, opera.
- **FAIL — mates requiring a sacrifice:** smothered (reached the Philidor setup,
  *stated the plan*, then flinched at the queen sac); blind-swine (blundered both
  rooks).

> The boundary: the agent converts material+technique mates but cannot execute
> mating combinations requiring a sacrifice, even with the forced line documented.

This is a clean Phase-1 result, not just a bug list. The deliberate-practice
infrastructure ([[skill-acquisition-loop]], [[ericsson1993deliberate]]) closes
the "won-but-drawn" gap that dominated the ranked data for *technique* mates —
exactly the K+Q / K+R / back-rank cases. What it does not yet buy is the
willingness to give up material for a forced mate; documenting the line is not
enough to make a 31B reader pull the trigger on a queen sacrifice. That
technique-yes / sacrifice-no line is worth carrying into the write-up.
