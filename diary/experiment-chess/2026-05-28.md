# 2026-05-28 — Closing the PR #1 calibration round

## Decision: this round is finished

PR #1 (`visualization-and-context-management`) is the first
infrastructure-improvement configuration on top of the bare-model
baseline. Its ranked batch on `main` has converged, so I'm closing this
round of calibration. Next step is a fresh branch for new skill work —
later, not now.

## The trajectory so far

| Configuration | Games | W–L–D | Elo |
|---|---|---|---|
| Baseline (model + `list_legal_moves` + `make_move`) | 26 | 0–22–4 | 1200 → **684.2** (≤ 700, pool-floor bounded) |
| PR #1 (perception tools + context mgmt + mate-seeking) | 27 | 14–8–5 | 684.2 → **793.6** |

The baseline number is a ceiling, not a point estimate
([[2026-05-25-baseline-calibration-complete]]). PR #1 climbed it by
~110 Elo and — more importantly — turned a 0-win configuration into one
that wins more than it loses (14–8–5) against a 700/850/1000 opponent
mix. That is the skill-acquisition signal the experiment exists to
measure: same model, better infrastructure, higher Elo.

## Why it's converged

The final batch (10 games) oscillates inside an ~25-point band
(786–812) with no trend — wins and losses trade off rather than the
rating drifting one way. Net 809.8 → 793.6. That's a rating that has
found its level for this configuration. More games at temp 1.0 buy
tighter error bars, not a different number.

If I want to report a single figure for PR #1 it's **~800 ± 15**, with
the honest framing that this is the configuration's level *as batched*.

## On the args parser bug

The ~20% `make_move` / 26% `imagine_move` failure rate documented in
[[2026-05-27-batch-analysis]] was a harness parser contract bug
(`shlex.split` choking on apostrophes in `--reasoning`, eating the
`--uci` flag), not a chess-reasoning failure. The fix landed after PR #1
merged (drop `--uci`/`--reasoning` flags, accept SAN+UCI, move
`run_script` to a `list[str]` args contract) and ran on a batch.

**I'm treating that fix as out-of-scope for the PR-as-configuration
unit.** It added no capability and gave the agent no new way to play —
it stopped well-formed tool calls from being silently rejected, i.e. it
stopped games from breaking. By the bug-fix-vs-configuration distinction
now written into [[experiment-chess]], a bug-fix restores the
configuration to measuring what it was supposed to measure; only a
capability change is a new datapoint on the trajectory. So the args fix
doesn't open a new round; the next *configuration* (new skill work on a
fresh branch) does.

Worth keeping in mind for the writeup: the PR #1 ~800 figure carries a
known tool-failure tax. The next configuration's number will reflect a
clean(er) contract, which is part of why I expect it to move.

## Standing reflections on this version (carried forward)

Recorded across [[2026-05-26-branch-wrapup]], [[2026-05-26-endgame-fixes]],
[[2026-05-27]], and [[2026-05-27-batch-analysis]]; consolidated here so
the round-close has the limitations in one place.

**What worked:**
- Perception tools were the decisive change — the attack/defense maps and
  hanging warnings map directly onto the mistakes the bare model made
  (miscounting attackers, missing defended pieces). Win-rate at low Elo
  jumped as soon as they landed.
- The mate-seeking prompt (enemy king mobility + "chase the mate when
  ahead") was the biggest behavioural unlock. Before it the agent would
  hold a 10-point lead for 60 moves without converting.
- Context management improved efficiency more than strength, but it's the
  infrastructure for later long-term-planning experiments.

**What still struggles:**
- King+rook mates — rarely finds the technique; wins by promoting to a
  second queen instead.
- Piece blunders despite warnings: (a) the hanging warning is value-blind
  on equal-count exchanges (the SEE-lite gap), and (b) when it does fire
  the agent sometimes commits anyway.
- Misses forks entirely (no fork detection exists).
- Non-immediate mates — even with the step-0 scan, misses mates that are
  "one defender away." First *concept-shaped* addition; brushes against
  the [[deterministic-tools-hypothesis]] line and needs careful sizing.

**Open structural question (deferred):** whether `skillful-agent` is
fighting Gemma 4 (malformed tool-call JSON, chain-of-thought channel
breakage). Leaning toward the modest fix — keep the harness, add
constrained decoding (`guided_json`) for tool args — rather than
replacing it, partly because the thesis itself is about infrastructure
and swapping the harness mid-experiment re-opens settled questions. Not
deciding in a diary entry.

**Unverifiable, parked:** the hanging-piece false-positive claim — zero
warnings fired across the analysed batch, so it can't be acted on
without a concrete reproduced instance.

## Next
